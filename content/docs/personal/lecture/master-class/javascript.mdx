## 런타임 메인 스레드

- 자바스크립트는 호출 스택이 단 하나만 존재한다,
- 동시에 여러가지 작업을 병렬로 처리할 수 없어서 웹워커를 사용해 병렬 처리가 가능하다.
- 스레드 안정성
  - 경쟁 조건이나 데드락 같은 문제가 발생하지 않는다.
  - 공유 자원 접근에 대한 복잡성 감소

## 경쟁 조건과 데드락

- 경쟁 조건: 여러 스레드/프로세스가 공유 자원에 동시에 접근하려 할 때, 실행 순서에 따라 결과가 달라지는 현상
- 데드락: 두 개 이상의 스레드/프로세스가 서로의 자원을 기다리며 영원히 블록되는 상태
  - 4가지 필요 조건: 상호 배제, 점유 대기, 비선점, 순환 대기
  - 문제점: 시스템이 정지된 것처럼 보인다. (외부 개입없이 해결되지 않음)

## 콜스택

- 원시타입 값을 저장
- 함수 호출 실행 컨텍스트 저장
- 요청이 동기적으로 실행

## 실행 컨텍스트

- 변수 객체, 스코프 체인, this의 정보를 담는다.

### this

- **현재 실행 컨텍스트(실행되는 환경)를 참조**하는 특별한 키워드
