# CS (Computer Science) 면접 질문

## 1. 객체 지향 프로그래밍에 대해 설명해 주세요. (중요도: 5)

**답변:**
객체지향프로그래밍(OOP, Object Oriented Programming)은 컴퓨터 프로그래밍 패러다임 중 하나로, 데이터를 추상화시켜 상태(attribute)와 행위(Method)를 가진 객체로 만들고 그 객체들 간의 상호작용을 통해 로직을 구성하는 프로그래밍 방법입니다.

**장점:** 코드 재사용 및 유지보수 용이, 대형 프로젝트에 적합  
**단점:** 처리속도 느림, 설계 시 많은 시간이 필요, 객체가 많을 시 용량이 커짐

**객체지향의 특성:**

1. **캡슐화**: 변수와 함수를 하나의 단위로 묶는 것, 정보 은닉
2. **상속화**: 이미 정의된 상위 클래스의 모든 속성과 연산을 하위 클래스가 물려받는 것
3. **추상화**: 객체들의 공통적인 특징을 도출하는 것
4. **다형성**: 하나의 변수 또는 함수가 상황에 따라 다른 의미로 해석될 수 있는 것

## 2. 프로세스와 스레드의 차이에 대해 설명해 주세요. (중요도: 5)

**답변:**
**프로세스(Process)**: 운영체제로부터 자원을 할당받는 작업의 단위  
**스레드(Thread)**: 할당받은 자원을 이용하여 실행되는 실행 흐름의 단위

프로세스 내의 스레드들은 Code, Data, Heap 영역을 공유하지만 Stack 영역은 각자 따로 가지고 있습니다.

---

## 3. 멀티 프로세스와 멀티 스레드의 차이에 대해 설명해 주세요. (중요도: 5)

**답변:**
**멀티 프로세스**: 하나의 프로그램을 여러 개의 프로세스로 나누어 각 프로세스가 하나의 작업을 처리하는 방식  
**멀티 스레드**: 하나의 프로그램을 여러 개의 스레드로 나누어 각 스레드가 하나의 작업을 처리하는 방식

**멀티 프로세스의 특징:**

- 장점: 안정성(하나의 프로세스가 죽어도 다른 프로세스에 영향을 끼치지 않음)
- 단점: 더 많은 메모리 공간과 CPU 시간을 차지

**멀티 스레드의 특징:**

- 장점: 시스템 자원 소모가 줄어듦, 시스템 처리량이 향상됨
- 단점: 디버깅이 까다로움, 하나의 스레드에 문제가 발생하면 전체 프로세스가 영향을 받음

## 4. Stack과 Queue의 차이에 대해 설명해 주세요. (중요도: 4)

**답변:**
**Stack (스택)**

- LIFO (Last In First Out) - 후입선출 방식
- 주요 연산: Push (삽입), Pop (삭제)
- 예시: 브라우저의 뒤로가기, 실행취소 기능

**Queue (큐)**

- FIFO (First In First Out) - 선입선출 방식
- 주요 연산: Enqueue (삽입), Dequeue (삭제)
- 예시: 프린터 출력 대기열, 은행 번호표

## 5. List, Map, Set의 차이점을 설명해 주세요. (중요도: 4)

**답변:**
**List**

- 순서가 있는 데이터들의 집합
- 중복 허용
- 인덱스로 데이터에 접근 가능

**Map**

- Key-Value 쌍으로 이루어진 데이터의 집합
- Key는 중복 불가, Value는 중복 가능
- Key를 통해 Value에 접근

**Set**

- 순서가 없는 데이터들의 집합
- 중복 허용하지 않음
- 집합 연산(교집합, 합집합 등) 가능

---

## 6. 라이브러리와 프레임워크의 차이점을 설명해 주세요. (중요도: 5)

**답변:**
**라이브러리 (Library)**

- 개발자가 필요한 기능을 선택적으로 호출하여 사용
- 제어의 주도권이 개발자에게 있음
- 예: jQuery, Lodash

**프레임워크 (Framework)**

- 정해진 프레임 안에서 개발자가 코드를 작성
- 제어의 주도권이 프레임워크에 있음 (제어의 역전, Inversion of Control)
- 전체적인 흐름을 프레임워크가 제어
- 예: React, Vue, Angular
